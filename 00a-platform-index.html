  • Platform Data Overview

  • Project entities are split into two core categories: 1.Devices and 2.Assets. Devices include all MEP equipment. Assets include buildings and each floor or main demise within each building
  • Each project entity has two data types: 1.Static/semi-static Attributes and 2.Timeseries Telemetry. Entities are mapped to Entities in the navigation menu
  • Static/semi-static data are data points that dont change or change infrequently. This could be a design parameter or a parameter that gets logged once per month
  • Timeseries data are data points that change frequently and are recorded at fixed time intervals, typically every 15 mintes. An example would be electrical power from a power meter

  • Project / Customer Navigation

  • Projects are mapped to Customers in the navigation menu. Each project name contains a SITE_NAME prefix and PROJECT_NNNN suffix. From there you can manage users, assets, devices and dashboards
  • Project assets are the buildng and the floors or demises within each building. Each asset name contains the NNNN identifier from the PROJECT_NNNN project ID
  • Project devices are all connected MEP equipment within the site. Each device name contains the NNNN identifier from the PROJECT_NNNN project ID
