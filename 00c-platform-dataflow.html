Site data transports specify how data is transmitted between site and the Attain platform. The platform uses simplified data structures to enable easy transmision of telemetry and command/control data

The Attain data transport design ensures maximum data efficiency and enables other outcomes such as rapid site onboarding and fast add-on with other apps and services

  • Site telemetry data object | SITE -- PUSH --> PLATFORM | 15 minute intervals
    • 
                
  • All device metadata and relations are stored as server side device attributes
    • 
              

            

              
    
                
  • Attain trim (commands) data object | SITE -- FETCH --> PLATFORM --> SITE | 15 minute intervals
    • 
                
    • 
                
  • Device commands are aggregated within each 15 minute interval then served from the Attain platform to the edge. Commands are then routed to applicable control devices
    • 
              

            

              
    
                
  • Attain fast updates (User engagement App commands) data object | SITE -- FETCH --> PLATFORM --> SITE | 1 minute interval
    • 
                
    • 
                
  • Device commands are aggregated within each 1 minute interval then served from the Attain platform to the edge. Commands are then routed to applicable control devices
    • 
              

            

              
    
                
  • Data from third party web services and pre-enabled buildings | ATTAIN VIRTUAL GATEWAY -- FETCH --> WEB SERVICE --> ATTAIN VIRTUAL GATEWAY -- PUSH --> PLATFORM
    • 
                
  • Data polling interval to be set depending on data requirements. Attain virtual gateway to map device names and telemetry key names to Attain standard before transporting data to platform
